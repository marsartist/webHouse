什么是跨域？

跨域又叫非同源策略请求，与之前相反的是同源策略请求（ajax / fetch）,

为什么会出现同源策略，非同源策略？

2013以前，前后端不分离，也就是前后端放在同一个服务器（也就是同一个域名，端口号，协议），所以不会有跨域问题。后来前后端进行分离，会出现前端和后端的代码并不部署在同一个web服务器上，也就是不在同一个域名下，即出现跨域。之所以有同源限制是为了安全考虑，防止不相关的网站发送请求操作自己的数据库。
服务器拆分：
    web服务器：静态资源
    data服务器：业务逻辑和数据分析
    图片服务器
同源（三者都一样，同源）
-协议
-域名
-端口号

1.什么是模块化？

模块化是指一个复杂的系统分解到多个模块以方便编码。

2.模块化的目的?

低耦合，可维护，命名空间，可复用

技术圈流传模块化规范有哪些？

命令空间，cmd,amd，commonjs，es6

命名空间 

a. 开发网页要通过命名空间的方式来组织的代码    
b. 命名空间冲突，两个库可能会使用同一个命名空间  
c. 无法合理的管理项目的依赖和版本
d.无法方便的控制依赖的加载顺序

AMD

  • 说明

    异步模块定义 Asynchronous Module Definition
    专门用于浏览器端、模块的加载是异步的

  • 优点

    a.可在不转换代码的情况下直接在浏览器中运行
    b.可加载多个依赖
    c.代码可运行在浏览器环境和node.js环境下

  • 缺点

    a. javascript运行环境没有原生支持AMD
    b. 需要先导入实现了AMD的库后才能正常使用

浏览器缓存

发表于 2019-12-24 分类于 网站性能优化

缓存一般针对get请求，对于post请求，浏览器一般不启用本地缓存。

1. 协商缓存

  • 什么是协商缓存？
    协商缓存其实是浏览器和的服务器商量着来（服务器告知浏览器可以缓存哪些内容？服务器告知浏览器是否可以使用本地缓存），当浏览器向web服务器请求一些内容的时，web服务器需要告诉浏览器哪些内容可以被缓存，
    一旦浏览器知道某个内容可以缓存后，下次当浏览器需要请求内容时，它便不会直接向服务器请求完整内容，而是询问服务器是否可以使用
    本地缓存，服务器在收到浏览器的询问后，需要作出回应，到底是允许浏览器使用本地缓存还是将最新的内容传回浏览器。

语义化

发表于 2019-12-24 分类于 html

1. 什么是语义化？

去掉用户自定义的样式后，页面仍呈现清晰结构（和浏览器的默认样式配合），比如<h>是粗体，字体大小是2em，（这里显示的是浏览器的默认样式）。即即使没有用户定义的样式，仍然知道清晰页面结构，比如标题。

2. 语义化解决了哪些问题？

  • 对于有视觉障碍的人阅读网站的时候，是通过阅读器。而阅读器实际上就是根据语义标签来识别文章的结构和内容。
  • pad、手机设备由于对css的支持比较弱，可能无法像浏览器一样渲染，而语义化的标签可以在一定的程度上的添加样式。
  • 语义化有利于seo，有助于seo网络蜘蛛爬取更多的信息，因为爬虫是根据标签来确定上下文和各个关键字的权重的。
  • 清晰的页面结构，有利于开发和维护。
