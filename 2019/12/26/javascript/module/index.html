模块化

发表于 2019-12-26 分类于 javascript ， 模块化

1.什么是模块化？

模块化是指一个复杂的系统分解到多个模块以方便编码。

2.模块化的目的?

低耦合，可维护，命名空间，可复用

技术圈流传模块化规范有哪些？

命令空间，cmd,amd，commonjs，es6

命名空间 

a. 开发网页要通过命名空间的方式来组织的代码    
b. 命名空间冲突，两个库可能会使用同一个命名空间  
c. 无法合理的管理项目的依赖和版本
d.无法方便的控制依赖的加载顺序

AMD

  • 说明

    异步模块定义 Asynchronous Module Definition
    专门用于浏览器端、模块的加载是异步的

  • 优点

    a.可在不转换代码的情况下直接在浏览器中运行
    b.可加载多个依赖
    c.代码可运行在浏览器环境和node.js环境下

  • 缺点

    a. javascript运行环境没有原生支持AMD
    b. 需要先导入实现了AMD的库后才能正常使用

    • 基本语法

      1. 定义暴露模块

        a.定义没有依赖的模块
     define(function(){
            return 模块;
      })     

b. 定义有依赖的模块
       define(['module1','module2',function(m1,m2){
            return 模块

        }]) ;

        1. 使用模块
          require([‘module1’,’module2’],function(m1,m2){

          使用m1,m2

          })

CMD

  • 说明

    模块的加载是异步的 Common Module Definition
    模块使用时才会加载执行

  • 基本语法

  • 实现

    浏览器端 Sea.js

COMMONJS

  • 说明

    每个文件都可以当做一个模块
    在服务端：模块的加载是运行时同步加载的
    在浏览器端：模块需要提前编译打包处理

  • 基本语法

    1、暴露模块
    module.exports = value
    exports.xxx = value
    2、引入模块
    require(xxx)

    xxx:
        a、第三方模块：xxx为模块名
        b、自定义模块：xxx为模块文件路径

  • 实现

    服务器端实现：node
    浏览器端实现：借助browserify

    区别node与browserify:
    a. nodejs运行时动态加载模块（同步)
    b. browerify是在转译时候就会加载打包合并

    原理：

    例如：

    a.js：
    module.exports=”你好世界”;
    b.js:

    const fs = require('fs');
  var req = function(moduleName){
  var content = fs.readFileSync(moduleName,'utf-8');
  var fn= new Function('module','exports','require','__dirname','__firename',content + 'return module.exports');
  return fn(module,module.exports,require,__dirname,__filename);

    }
    var val = req(“./a.js”);
    console.log(val);

ES6

  • 说明

    浏览器和服务器通用的模块解决方案
    依赖模块的需要编译打包处理

  • 基本语法

    1、导出模块 

    export

    2、引入模块

    import {} form 文件路经

  • 实现

    使用babel将es6转换成es5代码
    使用browserify编译打包js

  • 缺点

    es6模块虽然是终极模块化方案，但它的缺点在于目前无法直接运行在大部分javascript环境中，必须
    通过工具转换成标准的es5后才能正常运行。

  • 差异（与commonjs)

    commonjs模块输出的是一个值的拷贝，commonjs输出的是值的引用
    es6模块输出的是值的引用，commonjs编译时输出接口
