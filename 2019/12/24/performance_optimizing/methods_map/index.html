网站性能优化导图

发表于 2019-12-24 分类于 网站性能优化

思路：从浏览器输入url开始到浏览器显示完整网站全过程，思考可以进行优化点。

  1. 域名解析？

    • dns缓存
      • 操作系统中的dns客户端缓存的域名和ip地址信息。
      • dns服务器缓存的域名和ip地址的信息。

  2. 浏览器端（在发送请求，请求网站内容前）：查看浏览器缓存，或内存缓存。

    • 浏览器缓存
      • 强制缓存
      • 协商缓存
    • 内存缓存

  3. 网络：发送出去的请求经过路由器，到达指定的服务之前，可以做哪些优化呢？

    • 使用cdn(内容分发网络)

      什么cdn？

      内容发布网络是一组分布在多个不同地理位置web服务器，可以更有效的向用户发布内容。可以由cdn服务提供上提供，或者自己建立cdn。

      它是如何优化网络的？

      • 缩短响应时间（距离可能更近了）
        多台cdn服务器部署在不同的地方（相对于只有一台源服务器来说，cdn明显缩短了请求距离），当请求达到离客户端最近的的cdn服务器（边缘服务器）的时候，它会首先检查内容是否是最新的，缓存标识就是整个url地址。
如果内容被缓存，就返回的缓存内容。
如果内容没有被缓存，cnd服务器会向源服务器发送请求获取内容。
      • cdn具有备份、扩展存储功能、缓存

  4. 服务端：请求到达了服务器，那服务器端可以有哪些优化？

    • 响应的文件大小。

      • 压缩

      • 合并文件（适当就好，如果合并的文档过大，反而会影响http响应的时间）

        css sprites(css精灵)

        js合并（webpack、gulp这类打包工具）

    • 后端程序

      • 程序算法的执行效率

    • 数据库缓存

      • redis缓存
      • 临时表
      • 数据缓存
      • 定时任务

  5. 浏览器端（网络请求返回到浏览器后）：接下来就和浏览器如何dom树的构建，渲染页面，以及javascript执行效率有关了。

    • html构建dom树角度

      • 标签顺序

        外联样式最好写在head中

        外联脚本写在body标签里面底部

        • 如果放在head中，如果脚本有对dom树进行操作，会导致页面报错
        • 如果放在head中，脚本加载和执行会阻塞页面的解析，会影响效率

    • css渲染角度

      • 控制选择器叠加的个数。（过度的层叠选则的器会导致css解析变慢，且浏览器对css叠加的个数是有限制的）

    • javascript角度

      • javascript算法执行效率
      • javascript对dom树的操作（重排（回流），重绘）
        • 减少对dom树的操作，因为会导致页面重排
      • javascript对css样式的操作（重排（回流）、重绘）
        • 改变元素大小，宽度，高度，内边距，外边距，边框（会改变元素在文档流的位置）等会导致页面重排
        • 改变元素颜色，背景图片，字体颜色等不会的改变元素在文档流中的位置。
